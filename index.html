Foundation 5 Templates and Building Blocks

Sample Rendered Blog Post by John Smith,August 12, 2012.

This demo site is made with Zurb Foundation 5, and demonstrates a few different templates that could be used for different types of pages. There is also a "bare-bones" empty template (with no body and only the HTML <head> and the <script> tags at the end of the body).

If this were a real blog post, it could be a full post or a summary with a pre-set number of words. As it is, this blog post is somewhat "special" in that there's already some built-in dependencies on the CSS framework being used (hence these couple of paragraphs to the left of an image).

If this were a simple markdown file pre-rendered as HTML or dynamically rendered as HTML by JavaScript, then there would not be any multi-column layout, as above. Instead, it would render with the most basic of HTML that is generated from markdown.

Whether to use markdown or HTML is a matter of what you expect to author. HTML has many advantages in flexibility and content, but bring with it all the complexities and assumptions you might have to make in dynamically rendering it inside a given layout. Markdown has the advantage of simplicity, but is much more restricted on what can and cannot be rendered.

Below is a simple mustache-styled set of placeholders for a blog post. These are rendered inside an <article> tag.
